Клиническая задача: Определить, можно ли на основе доступных клинических данных (возраст, пол, ИМТ, курение, генетический риск и др.) предсказать вероятность наличия рака у пациента с высокой точностью.
Проверялась гипотеза о том, что комбинация факторов образа жизни (возраст, ИМТ, употребление алкоголя, курение) и генетической предрасположенности является значимым предиктором наличия рака.


Декомпозиция на шаги
Шаг 1. Подготовка окружения
Создать папку проекта: cancer-prediction-cnn
Инициализировать Git: git init
Создать виртуальное окружение: python -m venv venv
Установить зависимости: pip install pandas numpy scikit-learn tensorflow matplotlib seaborn
Шаг 2. Загрузка и анализ данных
Загрузить The_Cancer_data_1500_V2.csv
Визуализировать распределения 
Посчитать корреляции с Diagnosis
Шаг 3. Предобработка данных
Нормализовать числовые признаки (StandardScaler)
Разделить на X (признаки) и y (цель)
Разбить на train / test (например, 80/20)
Шаг 4. Построение модели CNN (1D)
Преобразовать X в форму (n_samples, n_features, 1) — как 1D-"изображение"
Шаг 5. Обучение и оценка
Обучить модель
Оценить: Accuracy, Precision / Recall, ROC-AUC
Шаг 6. Интерпретация и проверка гипотезы
Если AUC > 0.8 — гипотеза подтверждается: признаки содержат предиктивную информацию.
Если AUC ≈ 0.5 — гипотеза опровергается: текущие данные не позволяют предсказать рак.
Шаг 7. Оформление репозитория

ROC-AUC - это показатель того, насколько хорошо модель умеет разделять классы (рак/нет рака). 0.5 - случайное угадывание, 1.0 - идеальная модель.
Модель показала ROC-AUC = 0.7235. Это лучше случайного угадывания, но не идеально. Я делаю вывод, что гипотеза не подтверждается, так как выбранные признаки позволяют предсказывать наличие рака с вероятностью, незначительно выше случайной.
Модель хорошо работает с пациентами без рака (высокий recall = мало ложно отрицательных).
Плохо работает с пациентами, у которых есть рак (низкий recall = много ложных отрицательных - модель пропускает больных)
Для практического использования необходимо улучшить чувствительность модели. Например, за счёт балансировки классов или добавления дополнительных признаков.

Возможные ошибки в подборке методики:
CNN(Свёрточная нейронная сеть) обычно применяются к данным с пространственной структурой (изображения). Исходный датасет - табличные данные, и для них CNN не является оптимальным выбором. В коде представлен каждый "вектор" признаков как "одномерное изображение" (1D-CNN)
